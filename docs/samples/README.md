# 分析产出物样例（虚构项目 `order-platform`）

本目录为《历史 Java 工程分析系统设计》中约定交付物的**结构样例**。文件名采用层级序号：

| 层级 | 前缀 | 含义 |
|------|------|------|
| **0.x** | `0.1` | 仓库级总览（人读） |
| **1.x** | `1.1`、`1.2`… | 第一层：与仓库快照直接绑定的核心机器可读产物 |
| **2.x** | `2.1`、`2.2`… | 第二层：由第一层分解/派生的叶子报告与可选评估 |

## 文件一览

| 编号 | 文件 | 说明 |
|------|------|------|
| 0.1 | `0.1-REPOSITORY_OVERVIEW.md` | 仓库级总览与 Mermaid 架构草图 |
| 1.1 | `1.1-analysis_manifest.json` | 快照锚定（commit、JDK、Maven、解析器）与运行元数据 |
| 1.2 | `1.2-graph.json` | 有向依赖图；`manifest_ref` → `1.1` |
| 1.3 | `1.3-module_tree.json` | 分层模块树与叶子指标 |
| 1.4 | `1.4-entrypoints.json` | 入口点：REST、`main`、Quartz、SPI 等 |
| 2.1 | `2.1-leaf_order-service-core.md` | 典型**叶子模块**人读分析报告 |
| 2.2 | `2.2-rubric_eval.sample.json` | 可选：分层 rubric 叶子打分与加权聚合（示意） |

**推荐阅读顺序**：`1.1` → `0.1` → `1.2` / `1.3` / `1.4` → `2.1` →（可选）`2.2`。

数据为演示用途，与任何真实仓库无关。
